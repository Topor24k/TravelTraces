## About the Project
We are building a travel documentation web app focused on the Philippines. The app allows users to log and document every place they have visited — from famous tourist spots like Palawan and Siargao, to hidden local gems. Users can also explore and discover new destinations by browsing posts and travel stories shared by other users in the community.

## Tool Use (But Do Not Do This Yet, Until i Tell you So)
Tech stack: React (with Vite), TailwindCSS for styling, Node.js + Express for the backend, and PostgreSQL as the database. Use Supabase for authentication and storage. For maps, use Leaflet.js or Mapbox. Keep components modular and reusable. Prioritize mobile-first responsive design.

## What Should the App Do?
Key features to implement:
1. Travel Journal — users can create posts with photos, descriptions, ratings, and pin locations on a map of the Philippines.
2. Discovery Feed — a social feed where users can browse, like, and save travel posts from others.
3. Interactive Map — a visual map showing places the user has been, with color-coded pins.
4. Search & Filter — search destinations by region, category (beach, mountain, city), or tags.
5. User Profiles — each user has a profile showing their travel stats and visited places.
